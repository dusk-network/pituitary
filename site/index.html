Quick Start

30 seconds to first finding.

Single binary (pure Go, no CGO). No Docker. No API keys required. One SQLite file.

# macOS
brew install dusk-network/tap/pituitary

# Linux / macOS (auto-detect platform, verify checksum)
curl -fsSL https://raw.githubusercontent.com/dusk-network/pituitary/main/scripts/install.sh | sh

# Manual: download from GitHub Releases, then:
tar xzf pituitary_*_*.tar.gz
sudo install pituitary /usr/local/bin/
$ pituitary init --path . # discover, index, report
$ pituitary check-doc-drift --scope all # find stale docs
$ pituitary check-terminology --scope all # audit terminology
$ pituitary compile --dry-run # preview patches
$ pituitary review-spec --path specs/X # full composite review
$ git diff origin/main...HEAD | pituitary check-compliance --diff-file -

Deterministic by default. Optionally plug in any OpenAI-compatible embedding API for deeper semantic retrieval. Runtime docs →

You clean up after it, but the cleanup is another LLM session with the same partial view. The drift compounds.

Add the MCP server and your agent gets 13 governance tools backed by the full corpus index — not a context-window sample. It stops guessing and starts building against what you actually decided.

In CI

Catch contradictions before they reach main.

The shipped GitHub Action runs review-spec on PRs that touch specs and posts the report as a PR comment.

steps:
  - uses: dusk-network/pituitary@v1.0.0-beta.7
    with:
      fail-on: error

# Or add spec checks alongside your linter:
  - run: pituitary index --rebuild
  - run: git diff origin/main...HEAD | pituitary check-compliance --diff-file -
  - run: git diff origin/main...HEAD | pituitary check-doc-drift --diff-file -

Architecture

Deterministic core. No embedded agent.

Single SQLite file with atomic rebuilds. CLI-first — the MCP server wraps the same analysis packages. Kernel stays pure; extensions register via sdk.Register() and compile into one binary.
